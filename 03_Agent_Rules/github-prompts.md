# 📝 Примеры промптов для работы с GitHub

Практические примеры промптов для работы с Git-репозиторием на основе этого проекта.

## 🌿 Работа с ветками

### Создание новой ветки для функции
```
Контекст: Работаю над репозиторием github-knowledge-base
Задача: Создать новую ветку для добавления раздела о CI/CD
Требования:
- Ветка должна ответвляться от main
- Имя ветки в формате feature/<название>
Пример: feature/cicd-setup
```

### Переключение между ветками
```
Контекст: Нахожусь в ветке feature/basics-and-setup, нужно переключиться на main
Задача: Переключиться на основную ветку и показать текущий статус
Требования:
- Сохранить незакоммиченные изменения, если есть
- Показать разницу между ветками
```

## 💾 Коммиты и история

### Написание сообщения коммита
```
Контекст: Изменил файлы 01_Basics/README.md и 01_Basics/setup.md
Задача: Написать понятное сообщение коммита
Требования:
- Формат: <тип>: <описание>
- Тип изменения: docs, feat, fix, refactor
- Краткое описание до 50 символов
Примеры изменений:
- Добавлена структура раздела Основы
- Заполнен файл настройки окружения
```

### Просмотр истории изменений
```
Контекст: Нужно понять, какие изменения были внесены за последнюю неделю
Задача: Показать историю коммитов с деталями
Требования:
- Последние 10 коммитов
- Показать изменённые файлы в каждом коммите
- Формат: краткое описание + дата
```

### Отмена изменений
```
Контекст: Сделал коммит с ошибкой в сообщении или лишними файлами
Задача: Отменить последний коммит, сохранив изменения в рабочей директории
Требования:
- Не терять изменения в файлах
- Возможность перекоммитить с правильным сообщением
```

## 📤 Пуш и синхронизация

### Отправка изменений в удалённый репозиторий
```
Контекст: Локальные изменения закоммичены в ветке feature/new-section
Задача: Отправить изменения на GitHub
Требования:
- Ветка ещё не существует на удалённом сервере
- Нужно установить отслеживание удалённой ветки
```

### Синхронизация с upstream
```
Контекст: Моя ветка feature/docs-update отстала от main на несколько коммитов
Задача: Обновить ветку последними изменениями из main
Требования:
- Использовать rebase вместо merge для чистой истории
- Разрешить конфликты слияния, если возникнут
```

## 🔀 Pull Requests

### Подготовка к созданию PR
```
Контекст: Завершил работу над функцией в ветке feature/agent-rules
Задача: Подготовить ветку к созданию pull request
Требования:
- Проверить, что ветка актуальна относительно main
- Показать список изменённых файлов
- Предложить название и описание для PR
```

### Рефакторинг кода
```
Контекст: Файл 02_Workflow/README.md стал слишком большим
Задача: Предложить структуру для разделения на модули
Требования:
- Выделить отдельные темы в подфайлы
- Сохранить навигацию через оглавление
- Обновить ссылки в других файлах
```

## 🔍 Поиск и анализ

### Поиск по истории
```
Контекст: Нужно найти, когда и кем был добавлен файл setup.md
Задача: Найти коммиты, связанные с этим файлом
Требования:
- Показать автора, дату и сообщение коммита
- Показать изменения в файле
```

### Анализ конфликтов
```
Контекст: При merge возник конфликт в файле README.md
Задача: Помочь разрешить конфликт слияния
Требования:
- Показать конфликтующие секции
- Предложить варианты разрешения
- Сохранить полезные изменения из обеих версий
```

## 🛠️ Исправление ошибок

### Отмена неудачного merge
```
Контекст: Сделал merge не той ветки, нужно отменить
Задача: Отменить слияние и вернуться к состоянию до merge
Требования:
- Не потерять другие изменения
- Вернуть HEAD к коммиту до слияния
```

### Исправление пути коммита
```
Контекст: Закоммитил в неправильную ветку (нужно было в feature, а коммит в main)
Задача: Переместить коммит в правильную ветку
Требования:
- Удалить коммит из main
- Применить коммит к ветке feature
```

## 📊 Статистика и отчётность

### Статистика вклада
```
Контекст: Нужно подготовить отчёт о проделанной работе за спринт
Задача: Показать статистику изменений за период
Требования:
- Количество коммитов
- Изменённые файлы
- Строки добавлено/удалено
- Период: последние 2 недели
```

### Подготовка changelog
```
Контекст: Готовлю релиз v1.0, нужно составить список изменений
Задача: Сгенерировать changelog на основе коммитов
Требования:
- Группировка по типам: Features, Docs, Fixes
- Ссылки на коммиты
- Период: все коммиты с момента последнего релиза
```

## 🎯 Шаблоны промптов

### Для ежедневной работы
```
Контекст: [Текущая ветка и задача]
Задача: [Что нужно сделать прямо сейчас]
Изменения: [Какие файлы затронуты]
Требования: [Специфические требования]
```

### Для код-ревью
```
Контекст: [Описание изменения]
Задача: Провести код-ревью изменений
Файлы: [Список изменённых файлов]
Критерии:
- [Критерий 1]
- [Критерий 2]
```

### Для решения проблем
```
Контекст: [Описание проблемы]
Ошибка: [Текст ошибки илиunexpected behavior]
Что уже пробовал: [Предпринятые действия]
Окружение: [OS, версия Git, etc.]
```
