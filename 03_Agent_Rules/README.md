# 🤖 ИИ-Агент: Правила промптов

Гайдлайны для эффективной работы с ИИ-агентами при разработке и работе с кодом.

## 📋 Основные принципы

### 1. Будьте конкретны
- Чётко формулируйте задачу
- Указывайте ожидаемый результат
- Приводите примеры, если это уместно

**Плохо:**
> «Сделай лучше»

**Хорошо:**
> «Добавь обработку ошибок в функцию login() с выводом понятного сообщения пользователю»

### 2. Предоставляйте контекст
- Указывайте язык программирования и фреймворки
- Описывайте текущую архитектуру
- Прикрепляйте релевантные фрагменты кода

### 3. Разбивайте сложные задачи
- Декомпозируйте большие задачи на подзадачи
- Решайте проблемы последовательно
- Проверяйте каждый шаг перед переходом к следующему

### 4. Указывайте ограничения
- Требования к производительности
- Совместимость с определёнными версиями
- Предпочтения по стилю кода

## 🎯 Структура эффективного промпта

```
1. Контекст: [Описание проекта/проблемы]
2. Задача: [Что нужно сделать]
3. Требования: [Список обязательных условий]
4. Ограничения: [Что нельзя использовать/делать]
5. Примеры: [Примеры кода или поведения, если есть]
```

### Пример хорошего промпта:

```
Контекст: React-приложение с TypeScript, использую React Query для работы с API
Задача: Создать хук для загрузки данных пользователя
Требования:
- Типизация с TypeScript
- Обработка состояний загрузки и ошибок
- Кэширование на 5 минут
Ограничения: Не использовать Redux
Пример: [фрагмент существующего хука]
```

## ⚠️ Частые ошибки

| Ошибка | Как исправить |
|--------|---------------|
| Слишком общий запрос | Добавьте детали и контекст |
| Отсутствие примеров | Приведите пример желаемого результата |
| Игнорирование ограничений | Укажите технические ограничения заранее |
| Множественные задачи в одном промпте | Разбейте на отдельные запросы |

## 🔧 Специальные команды

- **«Объясни как...»** — для получения инструкций
- **«Найди ошибку...»** — для отладки кода
- **«Оптимизируй...»** — для улучшения производительности
- **«Напиши тесты...»** — для создания тестов
- **«Рефакторинг...»** — для улучшения структуры кода
