# 🌿 Ветвление и Pull Request

## Зачем нужны ветки?

Ветки (branches) позволяют изолировать изменения от основной кодовой базы (`main`). Это необходимо для:
- Разработки новых функций без риска сломать рабочую версию.
- Исправления багов в отдельном окружении.
- Параллельной работы нескольких разработчиков (или агентов) над разными задачами.
- Безопасного тестирования изменений перед слиянием.

## Как создавать ветки

### 1. Создание новой ветки
Чтобы создать новую ветку и сразу переключиться на неё:
```bash
git checkout -b <имя-ветки>
# Или современный вариант:
git switch -c <имя-ветки>
```
*Пример:* `git switch -c feature/add-login`

### 2. Переключение между ветками
```bash
git checkout <имя-ветки>
# Или:
git switch <имя-ветки>
```

### 3. Просмотр списка веток
```bash
git branch
```
Текущая ветка будет отмечена звездочкой `*`.

## Рабочий процесс (Workflow)

Наш стандартный процесс разработки выглядит так:

1. **Синхронизация с main**:
   ```bash
   git checkout main
   git pull origin main
   ```

2. **Создание ветки под задачу**:
   ```bash
   git switch -c fix/typo-in-readme
   ```

3. **Внесение изменений и коммит**:
   ```bash
   git add .
   git commit -m "fix: исправлена опечатка в README"
   ```

4. **Отправка ветки на сервер**:
   ```bash
   git push -u origin fix/typo-in-readme
   ```

## Как сделать Pull Request (PR)

Pull Request — это запрос на слияние ваших изменений из ветки разработки в основную ветку (`main`).

### Шаги создания PR:
1. Перейдите в репозиторий на GitHub.
2. Вы увидите уведомление о недавно запушенной ветке с кнопкой **"Compare & pull request"**.
3. Нажмите её, чтобы открыть страницу создания PR.
4. Заполните описание:
   - **Заголовок**: Кратко опишите суть изменений (например, "Добавлена инструкция по ветвлению").
   - **Описание**: Детали реализации, скриншоты (если нужно), ссылка на задачу.
5. Выберите ревьюеров (если требуется).
6. Нажмите **"Create pull request"**.

### Слияние (Merge)
После проверки изменений (код-ревью) и успешного прохождения тестов:
- Нажмите **"Merge pull request"**.
- Подтвердите слияние кнопкой **"Confirm merge"**.
- Удалите ветку после слияния (кнопка **"Delete branch"**), чтобы поддерживать чистоту репозитория.

> 💡 **Совет**: В нашем процессе ИИ-агент создает ветку, вносит правки и инициирует PR для утверждения изменений человеком.
