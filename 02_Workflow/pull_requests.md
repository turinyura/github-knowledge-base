# 🔄 Pull Requests (PR)

Pull Request — это механизм предложения изменений в репозиторий. Он позволяет обсудить код, проверить его и интегрировать в основную ветку.

## 🎯 Зачем нужны PR?

- **Code Review**: Возможность для команды проверить изменения перед слиянием.
- **Обсуждение**: Комментарии к конкретным строкам кода или общему решению.
- **CI/CD**: Автоматический запуск тестов и проверок перед мержем.
- **История**: Четкая фиксация того, какие изменения и зачем были внесены.

## 📝 Процесс создания PR

1. **Создайте ветку** от актуальной `main`:
   ```bash
   git checkout main
   git pull origin main
   git checkout -b feature/my-new-feature
   ```

2. **Внесите изменения** и закоммитьте их:
   ```bash
   git add .
   git commit -m "feat: добавлена новая функциональность"
   ```

3. **Отправьте ветку на сервер**:
   ```bash
   git push origin feature/my-new-feature
   ```

4. **Создайте PR на GitHub**:
   - Перейдите в репозиторий на GitHub.
   - Нажмите кнопку **"Compare & pull request"**.
   - Убедитесь, что базовая ветка — `main`, а сравниваемая — ваша.
   - Заполните описание: что сделано, зачем, какие тесты пройдены.
   - Назначьте ревьюеров (если требуется).

## 🤖 Пример из нашего процесса

В нашем рабочем процессе ИИ-агент создает ветку для каждой новой задачи:
```bash
git checkout -b agent/task-123-add-docs
```
После внесения правок агент пушит изменения и создает PR с подробным описанием изменений. Это позволяет человеку быстро понять суть правок и одобрить их.

## ✅ Чек-лист перед мержем

- [ ] Код проходит все проверки CI.
- [ ] Получено одобрение хотя бы одного ревьюера.
- [ ] Конфликты слияния разрешены.
- [ ] Описание PR понятное и полное.
