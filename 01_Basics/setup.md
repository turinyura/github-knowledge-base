# ⚙️ Настройка окружения

Перед началом работы с Git и GitHub необходимо правильно настроить локальное окружение. Это обеспечит корректную идентификацию ваших изменений и безопасное соединение с удаленным репозиторием.

## 🎯 Зачем нужна правильная настройка?

Правильная настройка окружения решает следующие задачи:
- **Идентификация автора**: Git должен знать, кто вносит изменения (имя и email)
- **Безопасность**: SSH-ключи позволяют работать без ввода пароля
- **Удобство**: Глобальные настройки упрощают повседневную работу
- **Совместимость**: Единые стандарты настройки для всей команды

---

## 1. Установка Git

### Для чего это нужно
Git — это распределённая система контроля версий, которая должна быть установлена на вашем компьютере для работы с репозиториями.

### 🔧 Методы установки

#### Windows
Скачайте установщик с официального сайта [git-scm.com](https://git-scm.com/download/win) и следуйте инструкциям мастера установки. Рекомендуется оставить настройки по умолчанию.

**Что происходит при установке:**
- Устанавливается Git Bash (терминал с поддержкой Git-команд)
- Добавляется интеграция с проводником Windows
- Настраиваются переменные окружения

#### macOS
Если у вас установлен Homebrew, выполните в терминале:
```bash
brew install git
```
Или скачайте установщик с [git-scm.com](https://git-scm.com/download/mac).

**Альтернатива:** Можно установить через Xcode Command Line Tools:
```bash
xcode-select --install
```

#### Linux (Ubuntu/Debian)
```bash
sudo apt update
sudo apt install git
```

**Для других дистрибутивов:**
- Fedora/RHEL: `sudo dnf install git`
- Arch Linux: `sudo pacman -S git`

### ✅ Проверка установки
```bash
git --version
```
Должна отобразиться версия Git (например, `git version 2.43.0`)

## 2. Базовая конфигурация пользователя

### 🎯 Для чего это нужно
Git должен знать, кто делает коммиты. Эта информация будет сохранена в каждом коммите и позволит отслеживать авторство изменений. **Критически важно использовать тот же email, что и в аккаунте GitHub** — это свяжет ваши локальные коммиты с профилем на GitHub.

### 🔧 Методы работы

#### Установка имени и email
```bash
git config --global user.name "Ваше Имя"
git config --global user.email "your.email@example.com"
```

**Параметры команды:**
- `--global` — настройка применяется ко всем репозиториям пользователя
- Без `--global` — настройка только для текущего репозитория

#### Проверка настроек
```bash
git config --list
```

#### Просмотр конкретной настройки
```bash
git config user.name
git config user.email
```

#### Изменение настроек
Просто выполните команду `git config` снова с новыми значениями — они перезапишут старые.

## 3. Настройка SSH для GitHub

### 🎯 Для чего это нужно
Использование SSH позволяет:
- **Подключаться к GitHub без ввода пароля** при каждом пуше
- **Безопасно аутентифицироваться** с помощью криптографических ключей
- **Автоматизировать процессы** (скрипты, CI/CD)

SSH использует пару ключей: приватный (хранится у вас) и публичный (загружается на GitHub).

### 🔧 Методы работы

#### Генерация ключа
Выполните команду (замените email на свой):
```bash
ssh-keygen -t ed25519 -C "your.email@example.com"
```

**Параметры:**
- `-t ed25519` — тип ключа (современный и безопасный алгоритм)
- `-C "email"` — комментарий для идентификации ключа

**Вопросы при генерации:**
1. `Enter file in which to save the key` — нажмите `Enter` для пути по умолчанию (`~/.ssh/id_ed25519`)
2. `Enter passphrase` — можно ввести пароль для дополнительной защиты или нажать `Enter` для пропуска

#### Добавление ключа в агент SSH
Агент SSH хранит расшифрованные приватные ключи в памяти:
```bash
eval "$(ssh-agent -s)"
ssh-add ~/.ssh/id_ed25519
```

**Для автоматического запуска агента** добавьте в `~/.bashrc` или `~/.zshrc`:
```bash
if [ -z "$SSH_AUTH_SOCK" ]; then
  eval "$(ssh-agent -s)"
fi
```

#### Добавление ключа в GitHub

1. **Скопируйте содержимое публичного ключа:**
   - **Windows (PowerShell):** `cat ~/.ssh/id_ed25519.pub | clip`
   - **macOS:** `pbcopy < ~/.ssh/id_ed25519.pub`
   - **Linux:** `xclip -sel clip < ~/.ssh/id_ed25519.pub` или `cat ~/.ssh/id_ed25519.pub`

2. **Перейдите в настройки GitHub:**
   - Откройте https://github.com/settings/keys
   - Нажмите **New SSH key**

3. **Заполните форму:**
   - **Title:** Название для идентификации (например, "Work Laptop")
   - **Key type:** Оставьте "Authentication Key"
   - **Key:** Вставьте содержимое публичного ключа

4. **Сохраните:** Нажмите **Add SSH key**

#### Проверка соединения
```bash
ssh -T git@github.com
```

**Ожидаемый результат:**
```
Hi username! You've successfully authenticated, but GitHub does not provide shell access.
```

**Возможные проблемы:**
- `Permission denied` — ключ не добавлен в GitHub или не загружен в агент
- `Could not resolve hostname` — проблемы с сетью

## 4. Рекомендуемые глобальные настройки

### 🎯 Для чего это нужно
Эти настройки делают работу с Git более удобной и соответствуют современным стандартам.

### 🔧 Методы работы

#### Установка ветки по умолчанию в `main`
```bash
git config --global init.defaultBranch main
```
**Зачем:** При создании нового репозитория главная ветка будет называться `main` вместо устаревшего `master`.

#### Настройка редактора кода
Для VS Code:
```bash
git config --global core.editor "code --wait"
```

**Для других редакторов:**
- Sublime Text: `git config --global core.editor "subl -n -w"`
- Vim: `git config --global core.editor "vim"`
- Nano: `git config --global core.editor "nano"`

**Зачем:** Git будет открывать выбранный редактор для написания сообщений коммитов.

#### Автоматическая расцветка вывода
```bash
git config --global color.ui true
```
**Зачем:** Вывод команд Git будет цветным, что улучшает читаемость.

#### Настройка стратегии слияния
```bash
git config --global pull.rebase false
```
**Зачем:** По умолчанию `git pull` будет использовать merge вместо rebase, что безопаснее для начинающих.

#### Сокращение для популярных команд (алиасы)
```bash
git config --global alias.co checkout
git config --global alias.br branch
git config --global alias.ci commit
git config --global alias.st status
git config --global alias.unstage 'reset HEAD --'
git config --global alias.last 'log -1 HEAD'
```

**Использование:**
- `git co main` вместо `git checkout main`
- `git st` вместо `git status`

## 5. Проверка готовности

### 🔧 Финальная проверка

Убедитесь, что Git установлен и настроен:
```bash
git --version
git config --list
```

Проверьте SSH-соединение:
```bash
ssh -T git@github.com
```

### ✅ Чек-лист успешной настройки

- [ ] Имя и email настроены корректно
- [ ] SSH-ключ сгенерирован и добавлен в GitHub
- [ ] Проверка `ssh -T git@github.com` успешна
- [ ] Настроен предпочтительный редактор

## 🆘 Решение проблем

### Проблема: Git не найден
**Решение:** Переустановите Git и убедитесь, что он добавлен в PATH.

### Проблема: SSH не работает
**Решение:**
```bash
# Проверьте, запущен ли агент
eval "$(ssh-agent -s)"

# Перечитайте ключ
ssh-add ~/.ssh/id_ed25519

# Проверьте права на файлы
chmod 700 ~/.ssh
chmod 600 ~/.ssh/id_ed25519
chmod 644 ~/.ssh/id_ed25519.pub
```

### Проблема: Конфликт email
**Решение:** Убедитесь, что email в `git config` совпадает с email в аккаунте GitHub.
